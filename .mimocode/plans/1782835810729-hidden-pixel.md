# Migração: Render → Vercel (Ably + Upstash Redis)

## Arquitetura Atual vs Nova

```
ATUAL (Render)                         NOVA (Vercel)
─────────────                          ─────────────
client/  → React SPA (estático)        client/  → React SPA (estático, Vercel)
server/  → Express + Socket.io         api/     → Vercel Serverless Functions
         + Redis (Render)                       Upstash Redis (serverless)
                                               Ably (real-time, substitui Socket.io)
```

## Socket.io Events (mapeamento completo)

| Evento (Socket.io) | Método Novo | Destino |
|---|---|---|
| `create_room` → callback({roomId, room}) | `POST /api/rooms` | API route |
| `join_room` → callback({room}) | `POST /api/rooms` (com `action: "join"`) | API route |
| `vote` | `POST /api/rooms/[id]/vote` | API route |
| `reveal_cards` | `POST /api/rooms/[id]/reveal` | API route |
| `reset_votes` | `POST /api/rooms/[id]/reset` | API route |
| `room_update` (broadcast) | Ably channel `room:{id}` | Mensagem `room_update` |
| `disconnecting` (cleanup) | Ably Presence | Detecta saída via API |

## Fluxo Novo

1. **Criar/Entrar na sala**: Client faz `POST /api/rooms` → API atualiza Redis → Publica `room_update` no Ably → Client recebe via subscription
2. **Votar**: Client faz `POST /api/rooms/[id]/vote` → API atualiza Redis → Publica `room_update` no Ably
3. **Revelar/Resetar**: Mesmo padrão HTTP + Ably broadcast
4. **Desconectar**: Client usa Ably Presence para detectar saída → API limpa jogador do Redis

- `cors` (server — não necessário com API routes)
- `express` (server — substituído por API routes)

## Arquivos a Modificar

### `client/src/services/socket.ts` → `client/src/services/realtime.ts`

**Atual:**
```ts
import { io } from 'socket.io-client';
const socket = io(URL, { autoConnect: false });
```

**Novo:**
```ts
import Realtime from 'ably/promises';

const ably = new Realtime.Promise({ key: ABLY_KEY });
// Subscribe to room channel
export function subscribeToRoom(roomId: string, onUpdate: (room: RoomData) => void) {
  const channel = ably.channels.get(`room:${roomId}`);
  channel.subscribe('room_update', (msg) => onUpdate(msg.data));
  return () => channel.unsubscribe();
}
```

### `client/src/pages/Home.tsx`

**Mudanças:**
- Remover `socket.connect()` e `socket.emit('create_room', ...)`
- Substituir por `fetch('/api/rooms', { method: 'POST', body: ... })`
- Após criar sala, navegar para `/room/{roomId}`

### `client/src/pages/Room.tsx`

**Mudanças:**
- Remover `socket.on('connect', ...)` / `socket.on('room_update', ...)`
- Remover `socket.emit('join_room', ...)` / `socket.emit('vote', ...)` / etc.
- Substituir por:
  - `fetch('/api/rooms', { method: 'POST', body: JSON.stringify({ action: 'join', ... }) })`
  - `fetch('/api/rooms/[id]/vote', { method: 'POST', body: ... })`
  - `subscribeToRoom(roomId, handleRoomUpdate)` via Ably
- Remover referência a `socket.id` (usar `userId` do localStorage)

### `client/package.json`

- Adicionar: `ably`
- Remover: `socket.io-client`

### `server/` → Removido ou ignorado

O diretório `server/` não será mais usado. Pode ser removido ou mantido para referência.

### `package.json` (root)

Atualizar scripts e dependências para refletir a nova arquitetura.
